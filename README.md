# Linux Lab - Directory Navigation Using cd Command

## 🎯 Objective
The objective of this lab was to understand and practice directory navigation in a Linux environment using the `cd` (change directory) command. This is a fundamental skill required for efficiently working within a Unix/Linux system, especially in cybersecurity tasks.

---

## 🛠️ Skills Learned
- Using the `cd` command for navigation  
- Understanding relative and absolute paths  
- Moving between directories in the filesystem  
- Navigating to parent directories  

---

## 🔍 Steps Performed
In this lab, I accessed the cloud-based Linux environment and used the terminal to practice directory navigation. I started by moving into directories using relative paths, such as `cd Documents`, which allowed me to navigate based on my current location.

Next, I used absolute paths to directly access directories from the root (`/`). For example, I navigated using a full path like `/home/user/Documents`, which works regardless of the current directory.

I also practiced moving back in the directory structure using the `cd ..` command. This command helped me move one level up to the parent directory, improving my understanding of how directories are structured in Linux.

## 💡 Key Takeaways

- Relative paths depend on current location  
- Absolute paths start from root (`/`)  
- `cd ..` moves to the parent directory
